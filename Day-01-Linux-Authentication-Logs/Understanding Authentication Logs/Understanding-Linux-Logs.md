# Understanding Linux Logs

## What is a Log?

A log is a record of events generated by the operating system or applications.

Logs help SOC analysts answer:

- What happened?
- When did it happen?
- Who performed the action?
- Was it successful?
- Was it suspicious?

## Why are logs important?

Without logs, there is no evidence during an investigation.

## Main Linux Log Directory

```
/var/log
```

This directory stores operating system and application logs.

## Important Logs

| Log | Purpose |
|------|----------|
| auth.log | Authentication events |
| syslog | General system logs |
| kern.log | Kernel logs |
| boot.log | Boot events |
| dpkg.log | Package installation |
| apache2 | Web server logs |

## Key Learning

Every investigation starts by identifying available log sources.
